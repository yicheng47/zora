# Zora v0.1 — Architecture & Implementation Plan

## Reference

- **Spec:** `docs/arch/product_spec_v0.md`
- **Roadmap:** v0.1 Foundation

---

## Key Decisions

- **Embedding:** Ollama (`nomic-embed-text`, 768 dims) default, FTS5/BM25-only fallback when unavailable
- **Vector search:** sqlite-vec extension, statically linked via rusqlite
- **Async strategy:** `Mutex<Connection>` + `spawn_blocking` for all SQLite ops; WAL mode
- **Embedding abstraction:** sealed `AnyEmbedder` enum (not `dyn Trait`) — avoids async lifetime issues
- **Error handling:** `thiserror` per module, `anyhow` only in `main.rs`
- **Tool writes:** observe/remember trigger direct incremental reindex, not through file watcher
- **Deferred to v0.2:** temporal decay, MMR re-ranking, SSE transport, snapshot auto-generation, git auto-commit

## Architecture

### Module Dependency Graph

```
config.rs         (leaf)
error.rs          (leaf)
embed/mod.rs      (config, error)
embed/ollama.rs   (embed/mod, error)
store.rs          (config, error)
index.rs          (config, error, embed, store)
search.rs         (config, error, index)
watch.rs          (config, index, store)
server.rs         (store, index, search, error)
main.rs           (wires everything)
```

### Ownership Model

| Component | Shared via | Notes |
|-----------|-----------|-------|
| ZoraConfig | Clone | Immutable after construction, cheap (paths + numbers) |
| AnyEmbedder | `Arc<AnyEmbedder>` | Enum dispatch: `Ollama(OllamaEmbedder)` or `None` |
| Store | `Arc<Store>` | Stateless — root path + file I/O methods |
| Index | `Arc<Index>` | Owns `Mutex<Connection>` + embedder ref |
| Search | Clone | Holds only config weights, no state |
| ZoraServer | Clone | rmcp requires Clone; all fields are Arc or lightweight |

### Async Boundaries

| Operation | Sync/Async | Rationale |
|-----------|-----------|-----------|
| File I/O (Store) | Sync via `spawn_blocking` | Local FS, microsecond latency |
| SQLite queries | Sync inside `spawn_blocking` | `rusqlite::Connection` is not Send |
| Embedding HTTP | Async (reqwest) | Network I/O to Ollama |
| MCP tool handlers | Async | Required by rmcp |
| File watcher | Async (tokio task) | Background event loop |
| Chunking, scoring | Sync (pure functions) | No I/O |

### SQLite Schema

```sql
-- File metadata (change detection)
CREATE TABLE memories (
    id INTEGER PRIMARY KEY,
    path TEXT UNIQUE,
    content_hash TEXT,
    memory_type TEXT,
    last_modified TEXT
);

-- Chunked content
CREATE TABLE chunks (
    id INTEGER PRIMARY KEY,
    memory_id INTEGER REFERENCES memories(id),
    content TEXT,
    line_start INTEGER,
    line_end INTEGER
);

-- Full-text search
CREATE VIRTUAL TABLE chunks_fts USING fts5(content, content=chunks, content_rowid=id);

-- Vector search (when embeddings available)
CREATE VIRTUAL TABLE chunks_vec USING vec0(embedding float[768]);

-- Avoid re-embedding unchanged content
CREATE TABLE embedding_cache (
    content_hash TEXT PRIMARY KEY,
    embedding BLOB
);
```

## Implementation

### Phase 0 — Dependency fixes & shared types

| File | Action |
|------|--------|
| `Cargo.toml` | MODIFY — schemars 0.8→1.x, replace serde_yaml, add sqlite-vec + zerocopy, rusqlite `bundled-full` |
| `src/config.rs` | CREATE — `ZoraConfig`, `EmbeddingConfig`, `SearchConfig` |
| `src/error.rs` | CREATE — `StoreError`, `IndexError`, `SearchError`, `EmbedError`, `ZoraError` |

Verify sqlite-vec compiles with rusqlite 0.34: minimal `CREATE VIRTUAL TABLE` + insert + query test.

### Phase 1 — Store (markdown file I/O)

| File | Action |
|------|--------|
| `src/store.rs` | IMPLEMENT — `Store` struct, `init()`, `read_file()`, `list_files()`, `append_signal()`, `write_memory()`, `read_signals()` |
| `src/main.rs` | MODIFY — wire `Commands::Init` |

- `init()` creates `~/.zora/{identity,disposition,context,signal,.index}` + `.gitignore`
- `read_file()` parses YAML frontmatter with gray_matter
- `append_signal()` appends to daily `signal/YYYY-MM-DD.md` with `## HH:MM` headers
- `write_memory()` uses atomic write (temp file + rename)

### Phase 2 — Embed (Ollama integration)

| File | Action |
|------|--------|
| `src/embed/mod.rs` | IMPLEMENT — `AnyEmbedder` enum, `Embedding` type alias, `from_config()` |
| `src/embed/ollama.rs` | IMPLEMENT — `OllamaEmbedder`, POST to `localhost:11434/api/embed`, batch support, `check()` health probe |

- `from_config()` tries Ollama, falls back to `AnyEmbedder::None` with warning
- `check()` pings Ollama at startup for fast failure with helpful message

### Phase 3 — Index (SQLite + FTS5 + sqlite-vec)

| File | Action |
|------|--------|
| `src/index.rs` | IMPLEMENT — `Index` struct, schema init, `chunk_file()`, `reindex_all()`, `reindex_changed()`, `vector_search()`, `bm25_search()`, `stats()` |
| `src/main.rs` | MODIFY — wire `Commands::Index`, `Commands::Status` |

- Chunking: respect frontmatter, split on `##`, keep <400 tokens whole, fallback split on `\n\n`
- Skip all vector operations when `AnyEmbedder::None`
- Embedding cache: check content hash before calling embedder

### Phase 4 — Search (hybrid retrieval)

| File | Action |
|------|--------|
| `src/search.rs` | IMPLEMENT — `Search` struct, `query()`, `fuse_scores()` |
| `src/main.rs` | MODIFY — wire `Commands::Search` |

- Hybrid: 0.7 vector + 0.3 BM25, normalize each to [0,1] before weighting
- BM25-only mode when embedder is None
- Returns `Vec<SearchResult>` with text, score, source file, memory type, line range

### Phase 5 — Server (MCP tools)

| File | Action |
|------|--------|
| `src/server.rs` | IMPLEMENT — `ZoraServer`, 4 tool handlers, `impl ServerHandler` |
| `src/main.rs` | MODIFY — wire `Commands::Serve` (stdio transport) |

- Tool input types derive `JsonSchema` (schemars 1.x) + `Deserialize`
- Tool descriptions loaded via `include_str!("../prompts/*.md")`
- `zora_observe` / `zora_remember` trigger direct incremental reindex after file write
- `zora_reflect` returns reflect prompt + unprocessed signals

### Phase 6 — Watch (file watcher)

| File | Action |
|------|--------|
| `src/watch.rs` | IMPLEMENT — `spawn_watcher()` background tokio task |
| `src/main.rs` | MODIFY — start watcher in serve command |

- notify-debouncer-mini, 1.5s debounce
- Filter: `.md` files only, ignore `.index/` and `ZORA.md`
- Handles create, modify, delete events

### Phase 7 — CLI polish

| File | Action |
|------|--------|
| `src/main.rs` | MODIFY — startup checks, error messages, `Commands::Snapshot` |

- `serve` startup: verify store exists (or auto-init), check Ollama, run initial index, start watcher
- Clear errors: "Ollama not running", "No index found", "Run `zora init` first"

## Verification

- **Unit tests:** chunking logic, score fusion, frontmatter parsing (no external deps)
- **Integration tests:** full pipeline with mock embedder (deterministic vectors)
- **E2E:** `zora init` → `zora index` → `zora search "test"` → verify results
- **MCP test:** `zora serve` → connect from Claude Code → call all 4 tools
- **Watcher test:** add file while serving → search finds it within ~2s
- **Fallback test:** stop Ollama → verify BM25-only search works
